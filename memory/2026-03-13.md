1. 丰询问 heartbeat 是否每 30 分钟执行
2. 初步检查：crontab、systemd timers、pm2 均为空
3. 遗漏检查：OpenClaw 内部 cron 系统 (`~/.openclaw/cron/jobs.json`)

**根本原因:**
- 记忆健康检查任务配置了 `"wakeMode": "next-heartbeat"`
- 但 heartbeat 定时任务本身没有配置
- 导致**循环依赖**：记忆检查等 heartbeat 唤醒，heartbeat 不存在
- 结果：每天 8:00 的记忆检查从未执行

**修复方案:**
- 从 `jobs.json` 中删除 `"wakeMode": "next-heartbeat"` 行
- 让记忆健康检查独立运行，到点就执行

**配置变更:**
```json
// 修改前
{
  "jobId": "memory-health-check-daily",
  "wakeMode": "next-heartbeat"  // ❌ 删除此行
}

// 修改后
{
  "jobId": "memory-health-check-daily"
  // ✅ 独立运行
}
```

## 关键学习点

### 1. OpenClaw 定时任务位置
- **配置文件:** `~/.openclaw/cron/jobs.json`
- **不依赖:** 系统 crontab、systemd timers
- **教训:** 优先检查 OpenClaw 内部配置，而不是系统工具

### 2. `wakeMode` 使用原则
- **含义:** 任务到期后等待 heartbeat 会话唤醒才执行
- **适合:** 低优先级任务、批量合并、资源敏感型
- **不适合:** 时间敏感任务、健康检查、重要通知
- **原则:** 默认不要用 `wakeMode`，重要任务独立运行

### 3. 记忆系统健康检查
- **频率:** 每天 8:00 AM
- **检查项:**
  - memory/ 目录是否有昨日文件
  - MEMORY.md 是否正常
  - Git 同步状态
  - 有异常时通知丰
