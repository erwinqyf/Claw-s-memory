### 任务 3: 学习研究 ✅

**执行时间:** 00:15 - 00:25

**调研主题:** MCP (Model Context Protocol) — 2026 年现状

#### 核心发现

**MCP 是什么：**
- Anthropic 于 2024 年底推出的开放协议
- 标准化 LLM 与外部工具/数据源的连接方式
- 类似 USB 统一外设连接的概念，MCP 统一 AI 模型与外部世界的交互

**三大原始能力 (Primitives):**
1. **Resources** — 只读数据源（文件、数据库、API 响应）
2. **Tools** — LLM 可调用的函数（带 inputSchema）
3. **Prompts** — 可复用的 prompt 模板

**传输层:**
- stdio（本地进程）
- SSE（Server-Sent Events）
- WebSocket（双向通信）

**2026 年生态现状:**
- 2000+ 开源 MCP 服务器（npm）
- VS Code Copilot / Cursor / JetBrains AI 全部支持
- AWS Bedrock / Azure AI / GCP Vertex 提供托管端点
- LangGraph / CrewAI / AutoGen 使用 MCP 做工具调用
- OpenClaw 通过 `mcpServers` 配置块接入 MCP

**与 OpenClaw 的关系：**
- OpenClaw 在 `openclaw.json` 中通过 `mcpServers` 配置接入外部 MCP 服务器
- OpenClaw 本身也可作为 MCP 服务器（通过 mcpmarket.com 的 openclaw-connector）
- 当前 Claw 内置的 skills（weather、github、browser 等）本质上就是 MCP tools 的概念

#### 关键洞察
1. **MCP 不是替代 OpenClaw skills** — skills 是更高层的封装，底层可基于 MCP
2. **安全模型值得关注** — MCP 不强制认证，需自行实现
3. **工具数量有上限** — 过多 tools 会降低 LLM 推理质量（Over-exposing 陷阱）
4. **Stateless > Stateful** — 无状态服务器更适合规模化

#### 产出
- 学习笔记已写入本文件
- 未来可探索：自建 MCP server 扩展 Claw 能力（如直接连接本地数据库）
