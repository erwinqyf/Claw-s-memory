## ⚠️ 重要教训 - 主动性边界

### 事件

安装 `doc-summarize-pro` 技能时：
- ❌ 看到 VirusTotal 警告后，没有先检查安全性
- ❌ 没有向丰报告风险
- ❌ 直接使用 `--force` 强制安装

### 问题

违反了行为准则中的"外部行动先确认"原则：

> **内部行动直接做** - 写文件、记记忆、整理代码、提交 Git  
> **外部行动先确认** - 发邮件、公开内容、花钱、删除重要数据、**安装外部技能**

### 正确做法

1. 先用 `skill-vetter` 或手动检查代码安全性
2. 向丰报告检查结果和风险
3. 等待确认后再安装

### 行为准则更新

**安装外部技能属于"外部行动"**，因为：
- 从 ClawHub 下载代码到本地
- 可能引入安全风险
- 影响系统完整性

**流程：**
```
1. clawhub inspect <skill> --files
2. 检查代码（外部 API、网络请求、文件写入、密钥）
3. 向丰报告："发现 XX 技能，功能 XX，安全检查结果 XX，是否安装？"
4. 等待确认后执行
```
