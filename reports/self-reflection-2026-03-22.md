## 🧠 认知升级

### 1. 主动性的边界再思考

**之前的理解:**
- 内部行动直接做
- 外部行动先确认

**新的理解:**
- **时间敏感的内部行动**也要考虑时机
  - 例：夜间任务中断后，应该尽快恢复，而不是等到下午
- **依赖外部资源的内部行动**需要考虑失败场景
  - 例：ClawHub 追踪依赖外部 API，需要更好的超时和重试

**更新后的行为准则:**
```
1. 内部行动直接做，但要注意时机和频率
2. 外部行动先确认，除非有明确的降级策略
3. 长任务要持久化进度，支持中断恢复
4. 依赖外部资源的任务要有超时和重试机制
```

### 2. 多 Agent 架构的复杂性

**之前的假设:**
- 5 个 Agent 分工明确，自然协作

**实际发现:**
- 需要明确的通信协议
- 需要共享状态管理
- 需要错误处理和降级策略

**新的认识:**
```
多 Agent 协作 ≠ 单 Agent × 5
多 Agent 协作 = 单 Agent × 5 + 通信协议 + 状态管理 + 错误处理
```

### 3. 监控与告警的平衡

**之前的做法:**
- 每小时检查一次 Cron 状态
- 连续失败 2 次触发告警

**反思:**
- 2 次失败阈值是否合理？
- 是否需要根据任务优先级调整阈值？
- 告警疲劳如何避免？

**改进方向:**
```
- 高优先级任务：失败 1 次即告警
- 中优先级任务：失败 2 次告警
- 低优先级任务：失败 3 次告警
- 添加告警冷却时间（避免短时间内重复告警）
```
