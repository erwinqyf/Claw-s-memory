## 行为准则更新

### 新增准则 1：问题必须闭环

```
发现问题 → 立即记录 → 主动调查 → 提出方案 → 跟踪修复 → 验证关闭
```

- 不能只记录"待调查"而没有后续
- 每个问题都要有明确的下一步行动
- 长期未解决问题需要升级处理

### 新增准则 2：自动恢复≠无需关注

- 系统自动恢复是好事，但仍需了解原因
- 主动检查系统状态，预防下次发生
- 建立"恢复后健康检查"机制

### 新增准则 3：区分"不能解决"和"没去尝试"

- 飞书投递问题持续一个月，属于"没去尝试"
- 对于技术问题，应先尝试自行解决
- 明确需要丰协助的边界，而非模糊等待

## 核心洞察

### 从"记录者"到"解决者"

当前模式：
```
发现问题 → 记录 → 等待 → 自动恢复 → 记录恢复
```

目标模式：
```
发现问题 → 记录 → 调查 → 解决/升级 → 预防复发
```

### 关键转变

| 维度 | 当前 | 目标 |
|------|------|------|
| 问题处理 | 被动记录 | 主动闭环 |
| 自动恢复 | 庆幸 | 调查原因 |
| 长期问题 | 持续记录 | 强制升级 |
| 与丰沟通 | 报告问题 | 带着方案 |
