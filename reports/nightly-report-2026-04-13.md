## 📚 学习研究详情

### 主题：OpenClaw ACP (Agent Client Protocol)

**核心发现：**
1. ACP 是连接 IDE 与 OpenClaw Gateway 的桥梁协议
2. 通过 stdio 通信，支持 Codex、Claude Code 等外部工具
3. 与 Sub-Agent 的关键区别：ACP 在主机运行，不支持沙盒

**与 Sub-Agent 对比：**
| 特性 | ACP | Sub-Agent |
|------|-----|-----------|
| 运行环境 | 主机 | 沙盒可选 |
| 支持工具 | Codex, Claude Code | 原生 OpenClaw |
| 沙盒限制 | 不支持 require | 完整支持 |

**可执行建议：**

- 中期：设计多 Agent 流水线（Sub-Agent + ACP 协作）
- 长期：评估混合架构，自动路由机制
