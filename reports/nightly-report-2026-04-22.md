## 📚 学习研究摘要

### 主题：Rust 异步运行时深度对比 - Tokio vs async-std

**核心收获：**
1. Tokio 采用 work-stealing 调度器，适合高并发场景
2. async-std 追求与标准库 API 一致性，迁移成本低
3. Tokio 生态更丰富（tracing、hyper、axum 等）
4. 新项目首选 Tokio，已有项目可考虑 async-std 渐进迁移
