**核心内容:**

#### 4 种通信模式对比

| 模式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| sessions_send | 指令/汇报 | 简单直接、支持超时 | 同步阻塞、不适合大批量 |
| 共享文件系统 | 大批量数据 | 解耦、持久化 | 需要轮询、文件锁风险 |
| 事件驱动 | 广播通知 | 轻量级、适合定时任务 | 只能发 main 会话 |
| Git 媒介 | 知识沉淀 | 版本历史、冲突检测 | 延迟高、不适合频繁更新 |

#### 推荐架构

混合通信模式：
- 指令/汇报 → sessions_send
- 状态同步 → 共享文件 (shared/)
- 知识沉淀 → Git 提交
- 系统事件 → systemEvent

#### 通信协议约定

- 消息格式标准（version、from、to、correlationId 等）
- 响应超时约定（urgent: 30s, high: 60s, normal: 300s, low: 无超时）
- 错误处理约定（标准错误响应格式）

#### 设计原则

1. 明确所有权
2. 幂等性
3. 可追溯性
4. 降级策略
