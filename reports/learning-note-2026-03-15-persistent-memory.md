## 📋 技能概述

**核心功能:** 为任意 Agent 添加持久化记忆能力，使其能够：
- 记住 prior work（先前的工作）
- 在会话之间维护上下文
- 继续长周期工作流

**创建时间:** 2026-03-12  
**更新时间:** 2026-03-14  
**版本:** 1.0.1

---

## 🔍 技术分析

### 设计思路

这个技能解决的核心问题是：**Agent 如何在会话重启后保持连续性？**

当前 OpenClaw 的记忆架构：
```
会话层 → 索引层 → 本地持久层 (MEMORY.md + memory/*.md) → 远程持久层 (GitHub)
```

`persistent-agent-memory` 可能提供的增强：
1. **跨会话记忆检索** - 自动加载相关历史记忆
2. **工作流连续性** - 记录未完成的任务状态
3. **上下文维护** - 保持长期项目的上下文

### 与现有架构对比

| 特性 | 当前 Claw 架构 | persistent-agent-memory |
|------|---------------|------------------------|
| 记忆存储 | MEMORY.md + SQLite | 待确认（可能是专用 DB） |
| 检索方式 | QMD 混合检索 (BM25 + Vector) | 待确认（语义检索？） |
| 会话连续性 | 依赖手动加载 | 自动加载？ |
| 工作流状态 | 无专门机制 | 支持长周期工作流 |

## 💡 启发与思考

### 1. 记忆自动加载机制

**当前问题:** 每次会话需要手动读取 MEMORY.md 和最近的 memory/*.md

**可能的改进:**
```javascript
// 伪代码：会话启动时自动加载
async function onSessionStart() {
  const recentMemories = await memory.search({
    timeframe: '7d',
    limit: 10,
    relevanceThreshold: 0.7
  });
  const ongoingTasks = await memory.getOngoingWorkflows();
  
  // 自动注入到上下文
  context.inject(recentMemories);
  context.inject(ongoingTasks);
}
```

### 2. 工作流状态追踪

**当前问题:** 夜间任务状态记录在 `nightly-task-state.json`，但没有与记忆系统整合

**可能的改进:**
- 将任务状态作为"工作记忆"存储
- 支持查询"上次做到哪里了"
- 自动恢复中断的工作流

### 3. 记忆生命周期管理

**当前架构:** 每日记忆文件 + 月度/季度巩固

**可借鉴的设计:**
- 记忆衰减（temporal decay）- 已在用
- 记忆压缩（compaction）- OpenClaw 内置
- 记忆归档（archive）- 考虑实现

## 🪞 反思

**为什么研究这个技能？**
- 夜间自主任务需要跨会话连续性
- 当前记忆加载是手动的，不够自动化
- 希望 Agent 能更"自主"地继续未完成的工作

**与 Claw 记忆架构的关系：**
- 我们的架构更复杂（四层设计、GitHub 远程持久化）
- 但这个技能在"自动加载"和"工作流追踪"方面有启发
- 可以考虑选择性借鉴，而非完全替换
