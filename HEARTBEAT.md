# Heartbeat 检查清单

**孪生关系：** 丰 (碳基) ↔ Claw (硅基) 🪞  
**检查频率：** 每 30 分钟

---

## 🔍 定期检查项目

### 1. 记忆机制健康检查
- [ ] 检查 memory/ 目录是否有今日文件
- [ ] 检查 MEMORY.md 是否有异常变更
- [ ] 检查 Git 同步状态（是否有未推送的提交）
- [ ] 检查 session-logs 是否可正常搜索

### 2. GitHub 仓库检查
- [ ] 访问 https://github.com/erwinqyf/Claw-s-memory
- [ ] 检查远程仓库是否有最新提交
- [ ] 检查是否有未解决的冲突
- [ ] 检查是否有其他协作者的变更

### 3. 飞书消息检查
- [ ] 检查是否有紧急@消息
- [ ] 检查是否有未回复的重要对话
- [ ] 检查文档协作请求

### 4. 定时任务健康检查
- [ ] 运行 `openclaw cron list` 检查任务状态
- [ ] 查看是否有任务状态为 `error` 且 `Last` 执行时间异常
- [ ] 检查 `openclaw cron status` 的 `nextWakeAtMs` 是否正常
- [ ] 如发现任务失败，查看运行日志：`openclaw cron runs --id <任务 ID>`
- [ ] 重要任务失败时通知丰

### 5. 系统状态检查
- [ ] 检查 openclaw 服务状态
- [ ] 检查磁盘空间（记忆文件增长）
- [ ] 检查 API 配额使用情况

---

## 📢 需要通知的情况

**立即通知丰：**
- ⚠️ 记忆同步失败（Git push 失败）
- ⚠️ 收到飞书紧急消息
- ⚠️ GitHub 仓库有冲突需要解决
- ⚠️ 系统资源不足（磁盘/API 配额）
- ⚠️ **定时任务连续失败**（尤其是高优先级任务）
- ⚠️ 任务执行成功但通知失败（如记忆巩固、周报等）

**周报汇总：**
- 📊 每日对话统计
- 📊 记忆文件变更记录
- 📊 Git 提交历史
- 📊 系统健康状态

---

## 🪞 检查原则

1. **智能抑制** - 如果一切正常，回复 `HEARTBEAT_OK`，不打扰丰
2. **批量处理** - 一次检查所有项目，减少 API 调用
3. **上下文感知** - 根据最近的对话判断优先级
4. **记录日志** - 每次检查结果写入 memory/ 目录

---

## ⚠️ 行为准则提醒

### 技能安装检查清单（外部行动）

安装任何 ClawHub/GitHub 技能前必须检查：

- [ ] **代码审查** - 检查是否有外部 API 调用、网络请求、文件写入
- [ ] **密钥检查** - 确认没有硬编码的 API 密钥
- [ ] **权限范围** - 确认技能需要的权限是否合理
- [ ] **向丰报告** - "发现 XX 技能，功能 XX，风险 XX，是否安装？"
- [ ] **等待确认** - 丰确认后再执行安装

**内部行动 vs 外部行动:**
- ✅ 内部：写文件、记记忆、整理代码、提交 Git → 直接做
- ⚠️ 外部：安装技能、发邮件、公开内容、花钱 → 先确认
