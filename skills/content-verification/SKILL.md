# 内容真实性验证技能

## 核心原则

**宁可说"不知道"，也不要编造！**

## 验证流程

### 1. 来源验证（Source Verification）

**检查项：**
- [ ] 链接是否可访问（HTTP 200）
- [ ] 来源是否权威（行业媒体、官方公告、学术报告）
- [ ] 日期是否在合理范围内
- [ ] 作者/机构是否真实存在

**工具：**
```bash
# 检查 HTTP 状态码
curl -sI "URL" | head -1

# 检查域名信息
whois domain.com

# 检查页面内容
curl -s "URL" | grep -i "关键词"
```

**可信来源示例（语言服务行业）：**
- ✅ Slator (slator.com) - 行业权威媒体
- ✅ ProZ.com - 译员社区
- ✅ TAUS - 行业标准组织
- ✅ Common Sense Advisory - 市场研究
- ✅ 公司官方博客（RWS、SDL、Smartling 等）
- ❌ 无署名的内容农场
- ❌ 无法访问的链接

---

### 2. 数据交叉验证（Data Cross-Check）

**检查项：**
- [ ] 关键数据是否有多个独立来源支撑
- [ ] 数据是否合理（符合常识和行业认知）
- [ ] 是否有时间/地点/人物等具体细节
- [ ] 是否过于"完美"（如：恰好是整数、增长率过于整齐）

**红旗信号（可能编造）：**
- ⚠️ 单一来源，无其他媒体报道
- ⚠️ 数据过于精确（如：1.42 亿英镑，23% 增长）
- ⚠️ 无法找到原始报告/研究
- ⚠️ 引用"匿名消息人士"
- ⚠️ 链接是示例格式（如 example.com、domain.com/test）

---

### 3. 内容审查（Content Audit）

**检查项：**
- [ ] 核心主张是否有证据支撑
- [ ] 引用是否准确（逐字核对）
- [ ] 是否有断章取义
- [ ] 是否有过度解读

---

## 可信度评分

| 等级 | 标准 | 是否可发布 |
|------|------|-----------|
| **高** | 多个权威来源交叉验证，链接可访问，数据合理 | ✅ 可发布 |
| **中** | 单一权威来源，链接可访问，无明显红旗 | ⚠️ 需标注"单一来源" |
| **低** | 来源不明、链接失效、有红旗信号 | ❌ 不可发布 |

```markdown

YYYY-MM-DD HH:MM

## 验证对象
[资讯标题/内容摘要]

## 验证结果

### 来源验证
- 链接状态：✅ HTTP 200 / ❌ 失效 / ⚠️ 需人工核实
- 来源权威性：✅ 权威 / ⚠️ 一般 / ❌ 可疑
- 日期合理性：✅ 合理 / ⚠️ 存疑

### 数据验证
- 关键数据 1：✅ 已交叉验证 / ⚠️ 单一来源 / ❌ 无法验证
- 关键数据 2：...

### 红旗信号
- [ ] 无
- [ ] 单一来源
- [ ] 链接失效
- [ ] 数据过于完美
- [ ] 其他：____

## 可信度评分
**等级**：高/中/低

**理由**：[简要说明]

## 建议
- ✅ 可发布 / ⚠️ 需标注 / ❌ 不可发布
- [其他建议]
```

## 使用场景

1. **新闻汇总发布前** - 验证所有资讯链接和数据
2. **引用研究报告时** - 验证报告真实存在
3. **生成教程/指南时** - 验证步骤可执行
4. **回答事实性问题时** - 验证信息准确性

## 工具集成

**自动化验证脚本：**
```bash
# 批量检查链接状态
python3 scripts/verify-links.py input.md

# 检查来源权威性
python3 scripts/check-source-credibility.py domain.com

# 交叉验证数据
python3 scripts/cross-check-data.py "关键词"
```

**推荐工具：**
- curl - HTTP 状态检查
- wayback machine - 历史存档验证
- Google News - 多来源交叉验证
- SimilarWeb - 网站流量/权威性评估
