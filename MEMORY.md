## 架构决策

### 2026-05-18: 定时任务架构重构（v2）

**旧架构问题：** 9 个独立 cron 任务，每个都启动完整 LLM session
- 每天 9+ 次 LLM 启动 → 频繁超时
- 碎片化通知 → 信息噪音
- 脚本类任务不需要 LLM

**新架构：** 脚本 + 报告分离
- **脚本层**（静默执行）：夜间脚本、ClawHub追踪、行业监控 → 结果写 data/exec-results/
- **报告层**（投递飞书）：晨间(7:00)、午间(12:00)、晚间(22:00) → 读取脚本结果，统一通知
- 通知从 9+ 条/天降到 3 条

### 2026-05-17: 子代理统筹模式

- ✅ Claw 用 `sessions_spawn` 调度子代理，结果汇总后由我汇报给丰
- 所有 cron 任务 delivery=none，静默执行
