| 痛点 | 解决方案 | 实际价值 |

## ✨ 关键功能解析

### 1. 智能 Markdown 发布

**自动渲染机制：**
- `write` 和 `append` 操作自动将 Markdown 转换为飞书结构化文档
- 支持标题、列表、粗体、斜体、代码、引用、分隔线
- 表格自动转换为格式化列表（飞书 API 不支持原生表格）

**最佳实践：**
```markdown
# 项目报告

## 关键指标
- **收入**: $100K
- **用户**: 10K

> 重要提示
> 这是引用块
```

### 2. 权限管理

**支持的操作：**
- 添加/删除协作者
- 更新权限级别（查看/编辑/完全访问）
- 列出现有权限
- 转移文档所有权

**权限级别：**
- `view` - 仅查看
- `edit` - 可编辑
- `full_access` - 完全访问

### 3. 重要区分：write/append vs update_block

| 特性 | write/append | update_block |
|------|--------------|--------------|
| Markdown 渲染 | ✅ 支持 | ❌ 纯文本 |
| 使用场景 | 初始内容、追加 | 快速文本更新 |
| 格式支持 | 完整 Markdown | 仅纯文本 |

**⚠️ 关键教训：** Markdown 内容始终使用 `write` 或 `append`，不要用 `update_block`

## 🔧 技术实现要点

### 必需权限
```json
{
  "scopes": {
    "tenant": [
      "docx:document",
      "docx:document:create",
      "docx:document:write_only",
      "docs:permission.member",
      "contact:user.base:readonly"
    ]
  }
}
```

### 支持的 Markdown 语法

| Markdown | 飞书效果 |
|----------|----------|
| `# 标题` | 标题1 |
| `## 标题` | 标题2 |
| `### 标题` | 标题3 |
| `- 项目` | 无序列表 |
| `1. 项目` | 有序列表 |
| `**粗体**` | 粗体 |
| `*斜体*` | 斜体 |
| `` `代码` `` | 行内代码 |
| `> 引用` | 引用块 |
| `---` | 分隔线 |

### 不支持的元素
- **表格** → 自动转换为列表
- **图片** → 需要单独上传
- **复杂 HTML** → 使用 Markdown
